## 十三、基础管理（档案/调度/菜单/日志/字典等）

> **功能路径**：`/system` 模块
> **权限要求**：租户管理员（tenant_admin）
> **套餐要求**：基础版及以上

### 13.1 基础档案

#### 13.1.1 功能描述
基础档案用于管理企业的通用业务档案数据，支持树形层级结构和标准 CRUD 操作。档案类型由管理员自定义，每种档案可维护独立的字段结构。

#### 13.1.2 数据表
档案分类表（archive_category）：

| 字段 | 类型 | 说明 |
|------|------|------|
| category_id | UUID | 主键 |
| tenant_id | UUID | 所属租户 |
| category_name | VARCHAR(50) | 分类名称 |
| category_code | VARCHAR(30) | 分类编码（唯一） |
| parent_id | UUID | 上级分类（NULL表示根） |
| icon | VARCHAR(50) | 图标 |
| sort_order | INT | 排序 |
| field_schema | JSONB | 字段定义 JSON Schema |
| status | ENUM | active / disabled |
| created_at | TIMESTAMPTZ | 创建时间 |

档案数据表（archive_record）：

| 字段 | 类型 | 说明 |
|------|------|------|
| record_id | UUID | 主键 |
| tenant_id | UUID | 所属租户 |
| category_id | UUID | 档案分类 |
| parent_id | UUID | 上级档案（树形） |
| record_code | VARCHAR(50) | 档案编码 |
| record_name | VARCHAR(100) | 档案名称 |
| data | JSONB | 档案数据（动态字段） |
| status | ENUM | active / disabled |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |

#### 13.1.3 API接口
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/tenants/{tenant_id}/archives/categories` | 获取档案分类树 |
| POST | `/api/v1/tenants/{tenant_id}/archives/categories` | 新建档案分类 |
| PUT | `/api/v1/tenants/{tenant_id}/archives/categories/{id}` | 更新档案分类 |
| DELETE | `/api/v1/tenants/{tenant_id}/archives/categories/{id}` | 删除档案分类 |
| GET | `/api/v1/tenants/{tenant_id}/archives/records` | 档案数据列表（分页+树形） |
| POST | `/api/v1/tenants/{tenant_id}/archives/records` | 新建档案数据 |
| GET | `/api/v1/tenants/{tenant_id}/archives/records/{id}` | 档案数据详情 |
| PUT | `/api/v1/tenants/{tenant_id}/archives/records/{id}` | 更新档案数据 |
| DELETE | `/api/v1/tenants/{tenant_id}/archives/records/{id}` | 删除档案数据 |

### UI规范

> **功能路径**：全局规范
> **权限要求**：所有用户
> **套餐要求**：所有套餐
> **规范版本**：V2.2（引用 `07_UI设计规范.md`）

#### 按钮命名规范

所有按钮必须使用「」双尖括号包裹中文名称，禁止使用英文或无包裹格式。

**正确示例：**
- 「搜索」「刷新」「新增用户」「编辑」「删除」「保存」「取消」「导出」「导入」

**错误示例：**
- ❌ `[搜索]`、`Search`、`search-btn`、`新增用户`

#### 按钮类型定义

| 按钮类型 | 视觉样式 | 使用场景 |
|----------|----------|----------|
| 主按钮 | 实心蓝色/品牌色 | 主要操作，如「保存」「提交」「通过」 |
| 次按钮 | 边框样式 | 次要操作，如「取消」「重置」「返回」 |
| 危险按钮 | 红色实心/边框 | 危险操作，如「删除」「禁用」「拒绝」 |
| 文字按钮 | 无边框 | 低强调操作，如「查看详情」「编辑」 |

#### 按钮权限要求

| 权限标识 | 说明 | 可见按钮 |
|----------|------|----------|
| all | 所有登录用户 | 搜索、刷新、查看详情 |
| tenant_admin | 租户管理员 | 含新建、编辑、删除、配置等全部写操作 |
| super_admin | 超级管理员 | 平台级管理，如删除租户、禁用用户 |

#### 按钮状态

| 状态 | 说明 | 视觉表现 |
|------|------|----------|
| normal | 默认状态 | 正常颜色，可点击 |
| hover | 悬停状态 | 颜色加深，显示手型光标 |
| loading | 加载状态 | 按钮禁用，显示加载图标 |
| disabled | 禁用状态 | 颜色变灰，不可点击 |

#### 页面布局（ArcoDesign Pro V2.2 列表页布局）

每个列表页面必须遵循以下统一的操作栏布局规范：

**布局结构：**
```

┌──────────────────────────────────────────────────────────┐
│  工作台 / 组织管理 / 公司管理        (面包屑，左上角)     │
│                                                          │
│  🔍 搜索...                         (搜索框，左对齐，280px)│
│                                                          │
│  「新建」「导出」「刷新」            (按钮组，左对齐)     │
│──────────────────────────────────────────────────────────│
│                        数据表格                          │
└──────────────────────────────────────────────────────────┘

```

**布局要点：**
1. **面包屑**：页面最左上角，格式：首页 / 模块名 / 页面名
2. **搜索框**：面包屑下方，左对齐，宽度约 280px，支持关键字筛选
3. **按钮组**：搜索框同一行下方（换行），**靠左排列**，顺序为「新建」→「导出」→「刷新」
4. **数据表格**：按钮组下方，占据页面剩余空间

**规则：**
- 面包屑在最顶部，显示当前页面路径
- 搜索框位于面包屑下方，左对齐，宽度约 280px
- 操作按钮（新建、导入、导出）放在搜索框下方，**靠左对齐**
- 按钮之间间距：8px
- 按钮高度：32px（标准）或 28px（紧凑）

#### 表格布局规范

| 元素 | 规范 |
|------|------|
| 列宽 | 最小 80px，最大自适应，超长文字省略 |
| 行高 | 标准 48px，紧凑 36px |
| 斑马纹 | 可选，建议复杂表格启用 |
| 复选框 | 多选表格第一列，宽度 40px |
| 操作列 | 固定右侧，最小宽度 120px |
| 空状态 | 居中显示图标+文字，无"暂无数据"纯文字 |

#### 弹窗规范（V2.2 新增详情弹窗规范）

| 弹窗类型 | 触发场景 | 尺寸 |
|----------|----------|------|
| 全屏模态（Modal） | 新建/编辑复杂表单（如新建租户、编辑用户） | 宽度 800~1200px，高度 80vh |
| 抽屉（Drawer） | 快速操作（如分配角色、筛选条件） | 宽度 400~600px |
| 确认对话框（Dialog） | 删除确认、状态切换确认 | 宽度 400px，固定居中 |
| 消息提示（Toast） | 操作结果反馈 | 居中浮现 |

**新建/编辑模态（全屏模态，风格D）：**
- 右上角：「×」关闭按钮 + 「保存」+「取消」
- 内容区：表单主体，滚动
- 底部：固定在模态框底部，「取消」在左，「保存」在右

**抽屉规范：**
- 从右侧滑入
- 右上角：「×」关闭按钮
- 宽度：400px（筛选）、500px（详情）

#### 交互规范

- **删除确认**：所有删除操作必须弹出确认对话框，显示待删除对象信息
- **Toast 反馈**：操作后顶部居中显示成功/失败提示，3~5秒自动消失
- **表单校验**：失焦时校验必填项和格式，错误信息显示在字段下方；提交时滚动到第一个错误处
- **加载状态**：首次加载显示骨架屏，提交时按钮显示 loading 禁止双击

#### 响应式规范（V2.2）

| 断点 | 屏幕宽度 | 布局变化 |
|------|----------|----------|
| 超大屏 | ≥1920px | 表格列全展示 |
| 大屏 | 1440~1919px | 标准布局 |
| 中屏 | 1024~1439px | 操作栏换行 |
| 小屏 | 768~1023px | 表格水平滚动 |
| 平板 | <768px | 强制收起操作栏 |

#### 列表页标准布局（V2.2 新增）

所有列表页必须遵循以下统一的页面布局规范：

**列表页标准布局图：**

```
□全选  │  编号  │  名称(可点击)  │  类型  │  状态  │  时间  │

新建    编辑    删除    │    批量导入    导出    🔄
```

**布局要点：**

1. **多选框**：每行第一列为复选框，配合表头全选框实现批量选择
2. **名称/id可点击**：名称列（和编号列）为可点击文本，点击后打开详情弹窗（Drawer），**不跳转到新页面**
3. **操作按钮分组**：
   - **左侧组**（写操作）：新建、编辑、删除
   - **右侧组**（数据操作）：批量导入、导出、刷新（🔄图标按钮）
   - 两组之间用分隔符 `│` 分隔
4. **行内无编辑/删除按钮**：编辑和删除操作统一放在顶部按钮组，不在表格每行提供操作按钮
5. **详情弹窗规范**：
   - 类型：Drawer（从右侧滑入，宽度 500px）
   - 弹窗内不提供编辑功能，查看即可
   - **Footer 只显示「取消」按钮**，无确认/保存按钮
6. **批量操作**：勾选多条记录后，顶部出现批量操作提示条（如"已选择 3 条记录"）

**按钮组示例：**

| 位置 | 按钮名称 | 说明 |
|------|----------|------|
| 操作栏（左） | 「新建」 | 打开全屏模态（新建表单） |
| 操作栏（左） | 「编辑」 | 选中一条记录后启用，打开全屏模态 |
| 操作栏（左） | 「删除」 | 选中记录后启用，弹出确认对话框 |
| 操作栏（右） | 「批量导入」 | 打开全屏模态（导入表单） |
| 操作栏（右） | 「导出」 | 导出 Excel 文件 |
| 操作栏（右） | 🔄（刷新） | 刷新列表数据 |
