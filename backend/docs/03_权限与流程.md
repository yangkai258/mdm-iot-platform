## 九、组织管理（公司/部门/岗位/员工）

### 9.1 公司管理

#### 9.1.1 实体字段



| 字段 | 类型 | 说明 |

|------|------|------|

| company_id | UUID | 主键 |

| tenant_id | UUID | 所属租户 |

### 9.3 岗位管理



岗位是介于用户和部门之间的组织维度，用于定义职责和权限模板。



#### 9.3.1 实体字段



| 字段 | 类型 | 说明 |

|------|------|------|

| post_id | UUID | 主键 |

| tenant_id | UUID | 所属租户 |

| dept_id | UUID | 所属部门（可空） |

| post_name | VARCHAR(50) | 岗位名称 |

| post_code | VARCHAR(20) | 岗位编码（租户内唯一） |

| description | VARCHAR(200) | 岗位描述 |

| permission_template | JSONB | 权限模板，如 `{"devices": ["read","control"], "ota": false}` |

| status | ENUM | active / disabled |

| created_at | TIMESTAMPTZ | 创建时间 |



#### 9.3.2 API



| 方法 | 路径 | 说明 |

|------|------|------|

| GET | `/api/v1/tenants/{tenant_id}/posts` | 岗位列表 |

| PUT | `/api/v1/tenants/{tenant_id}/posts/{id}` | 更新岗位 |

## 十、权限管理（角色/用户/数据权限）



> **功能路径**：`/admin/roles`、`/admin/permissions`

> **权限要求**：超级管理员（平台级角色）；租户管理员（租户内角色）

> **套餐要求**：基础版及以上



### 10.1 角色管理



#### 10.1.1 平台级角色（超级管理员维护）



| 角色标识 | 名称 | 描述 |

|----------|------|------|

| super_admin | 超级管理员 | 平台运营方，可管理所有租户、套餐、系统配置 |

| auditor | 审计员 | 仅可查看审计日志，不可操作业务数据 |



#### 10.1.2 租户内角色



| 角色标识 | 名称 | 描述 |

|----------|------|------|

| tenant_admin | 租户管理员 | 租户内最高权限，管理本单位所有功能 |

| store_manager | 门店管理员 | 管理指定门店/部门下的设备和数据 |

| operator | 操作员 | 使用设备功能，数据范围受限于所属部门 |

| viewer | 查看者 | 仅查看数据，无写操作权限 |



#### 10.1.3 角色实体字段



| 字段 | 类型 | 说明 |

|------|------|------|

| role_id | UUID | 主键 |

| tenant_id | UUID | 所属租户（平台级角色为 NULL） |

| role_name | VARCHAR(50) | 角色名称 |

| role_code | VARCHAR(30) | 角色代码（唯一） |

| description | VARCHAR(200) | 角色描述 |

| is_system | BOOLEAN | 是否系统内置（不可删除） |

| created_at | TIMESTAMPTZ | 创建时间 |



#### 10.1.4 API



| 方法 | 路径 | 说明 |

|------|------|------|

| GET | `/api/v1/admin/roles` | 角色列表 |

| POST | `/api/v1/admin/roles` | 新增角色 |

| GET | `/api/v1/admin/roles/{id}` | 角色详情（含权限列表） |

| PUT | `/api/v1/admin/roles/{id}` | 更新角色 |

| DELETE | `/api/v1/admin/roles/{id}` | 删除角色（is_system=false 可删） |

| GET | `/api/v1/admin/roles/{id}/permissions` | 获取角色权限 |

| PUT | `/api/v1/admin/roles/{id}/permissions` | 更新角色权限 |

### 10.5 数据权限维度

系统支持多层次的数据权限控制，不同维度可叠加生效。

#### 10.5.1 维度类型

| 维度 | 标识 | 说明 | 适用场景 |
|------|------|------|----------|
| 租户维度 | tenant | 强制隔离，所有业务数据按租户划分 | 所有用户 |
| 部门维度 | dept | 用户只能访问所属部门及下级数据 | 普通用户 |
| 门店维度 | store | 用户只能访问分配给他的门店数据 | 门店管理员 |
| 自定义维度 | custom | 管理员手动分配任意数据范围 | 特殊权限用户 |

#### 10.5.2 维度配置 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/tenants/{tenant_id}/data-dimensions` | 获取所有数据维度配置 |
| PUT | `/api/v1/tenants/{tenant_id}/data-dimensions/{dimension}` | 更新维度配置（启用/禁用/调整范围） |

**请求体**：

```json

{

  "enabled": true,

  "scope_type": "all" | "assigned" | "own",

  "assigned_scope": {

    "dept_ids": ["uuid1"],

    "store_ids": ["uuid2"]

  }

}

```

## 十一、流程管理（BPMN）



> **功能路径**：`/workflow`

> **权限要求**：租户管理员

> **套餐要求**：企业版



### 11.1 流程引擎概述



系统集成 Camunda BPMN 2.0 流程引擎，支撑业务流程的在线编排、审批与自动化执行。



### 11.2 流程定义



#### 11.2.1 流程分类



| 流程类型 | 英文标识 | 说明 |

|----------|----------|------|

| 设备入网审批 | device_onboarding | 新设备注册审批流 |

| 固件升级审批 | firmware_upgrade | OTA 升级申请审批流 |

| 优惠券发放审批 | coupon_approve | 大额优惠券发放审批 |

| 会员升级审批 | member_upgrade | 会员等级人工干预审批 |

| 数据导出审批 | data_export | 敏感数据导出审批流 |



#### 11.2.2 流程定义实体字段



| 字段 | 类型 | 说明 |

|------|------|------|

| process_def_id | UUID | 流程定义主键 |

| tenant_id | UUID | 所属租户 |

| process_key | VARCHAR(50) | 流程唯一标识 |

| process_name | VARCHAR(100) | 流程名称 |

| description | VARCHAR(200) | 流程描述 |

| bpmn_xml | TEXT | BPMN 2.0 XML 内容 |

| form_schema | JSONB | 表单 JSON Schema |

| variables_schema | JSONB | 流程变量定义 |

| status | ENUM | draft / published / archived |

| version | INT | 版本号 |

| created_at | TIMESTAMPTZ | 创建时间 |

| updated_at | TIMESTAMPTZ | 更新时间 |



#### 11.2.3 API



| 方法 | 路径 | 说明 |

|------|------|------|

### UI规范

> **功能路径**：全局规范
> **权限要求**：所有用户
> **套餐要求**：所有套餐
> **规范版本**：V2.2（引用 `07_UI设计规范.md`）

#### 按钮命名规范

所有按钮必须使用「」双尖括号包裹中文名称，禁止使用英文或无包裹格式。

**正确示例：**
- 「搜索」「刷新」「新增用户」「编辑」「删除」「保存」「取消」「导出」「导入」

**错误示例：**
- ❌ `[搜索]`、`Search`、`search-btn`、`新增用户`

#### 按钮类型定义

| 按钮类型 | 视觉样式 | 使用场景 |
|----------|----------|----------|
| 主按钮 | 实心蓝色/品牌色 | 主要操作，如「保存」「提交」「通过」 |
| 次按钮 | 边框样式 | 次要操作，如「取消」「重置」「返回」 |
| 危险按钮 | 红色实心/边框 | 危险操作，如「删除」「禁用」「拒绝」 |
| 文字按钮 | 无边框 | 低强调操作，如「查看详情」「编辑」 |

#### 按钮权限要求

| 权限标识 | 说明 | 可见按钮 |
|----------|------|----------|
| all | 所有登录用户 | 搜索、刷新、查看详情 |
| tenant_admin | 租户管理员 | 含新建、编辑、删除、配置等全部写操作 |
| super_admin | 超级管理员 | 平台级管理，如删除租户、禁用用户 |

#### 按钮状态

| 状态 | 说明 | 视觉表现 |
|------|------|----------|
| normal | 默认状态 | 正常颜色，可点击 |
| hover | 悬停状态 | 颜色加深，显示手型光标 |
| loading | 加载状态 | 按钮禁用，显示加载图标 |
| disabled | 禁用状态 | 颜色变灰，不可点击 |

#### 页面布局（ArcoDesign Pro V2.2 列表页布局）

每个列表页面必须遵循以下统一的操作栏布局规范：

**布局结构：**
```

┌──────────────────────────────────────────────────────────────────────┐
│  工作台 / 组织管理 / 公司管理              (面包屑，左上角)           │
│                                                                      │
│  🔍 搜索...                                   (搜索框，左对齐，280px)  │
│                                                                      │
│  「新建」  「编辑」  「删除」   │   「批量导入」  「导出」  🔄        │
│  (主操作按钮组，左对齐)          (辅助操作按钮组)                     │
│────────────────────────────────────────────────────────────────────│
│  □全选  编号  │ 名称(可点击) │ 类型    │ 状态  │ 时间    │ 操作     │
│────────────────────────────────────────────────────────────────────│
│  ☐        001  │ [项目名称]  │ 运行中 │ ●在线 │ 2小时   │          │
│  ☐        002  │ [项目名称]  │ 已停止 │ ○离线 │ 3天     │          │
└────────────────────────────────────────────────────────────────────┘

```

**布局要点：**
1. **面包屑**：页面最左上角，格式：首页 / 模块名 / 页面名
2. **搜索框**：面包屑下方，左对齐，宽度约 280px，支持关键字筛选
3. **按钮分组**：
   - **主操作按钮组**（左）：「新建」「编辑」「删除」紧靠在一起
   - **辅助操作按钮组**：「批量导入」「导出」「🔄刷新」挨在一起
   - 两组之间用 `│` 分隔
4. **数据表格**：
   - 每行左侧有复选框（支持全选），宽度 40px
   - **名称/id 可点击**：点击名称或编号打开详情弹窗，不在表格行内显示编辑/删除按钮
   - 操作列不再固定显示编辑/删除
5. **详情弹窗**：footer 只显示「取消」按钮

**规则：**
- 面包屑在最顶部，显示当前页面路径
- 搜索框位于面包屑下方，左对齐，宽度约 280px
- 主操作按钮：「新建」「编辑」「删除」靠左排列，间距 8px
- 辅助操作按钮：「批量导入」「导出」「🔄刷新」靠在一起
- 按钮之间间距：8px
- 按钮高度：32px（标准）或 28px（紧凑）

#### 表格布局规范

| 元素 | 规范 |
|------|------|
| 列宽 | 最小 80px，最大自适应，超长文字省略 |
| 行高 | 标准 48px，紧凑 36px |
| 斑马纹 | 可选，建议复杂表格启用 |
| 复选框 | 多选表格第一列，宽度 40px |
| 操作列 | 固定右侧，最小宽度 120px |
| 空状态 | 居中显示图标+文字，无"暂无数据"纯文字 |

#### 弹窗规范（V2.2 新增详情弹窗规范）

| 弹窗类型 | 触发场景 | 尺寸 |
|----------|----------|------|
| 全屏模态（Modal） | 新建/编辑复杂表单（如新建租户、编辑用户） | 宽度 800~1200px，高度 80vh |
| 抽屉（Drawer） | 快速操作（如分配角色、筛选条件） | 宽度 400~600px |
| 确认对话框（Dialog） | 删除确认、状态切换确认 | 宽度 400px，固定居中 |
| 消息提示（Toast） | 操作结果反馈 | 居中浮现 |
| **详情弹窗** | 点击名称/id打开详情 | 宽度 600~800px，仅显示信息 |

**新建/编辑模态（全屏模态，风格D）：**
- 右上角：「×」关闭按钮 + 「保存」+「取消」
- 内容区：表单主体，滚动
- 底部：固定在模态框底部，「取消」在左，「保存」在右

**抽屉规范：**
- 从右侧滑入
- 右上角：「×」关闭按钮
- 宽度：400px（筛选）、500px（详情）

**详情弹窗（V2.2 新增）：**
- 右上角：「×」关闭按钮
- 内容区：只读展示信息，滚动
- 底部：**仅显示「取消」按钮**，无保存/编辑按钮

#### 交互规范

- **删除确认**：所有删除操作必须弹出确认对话框，显示待删除对象信息
- **Toast 反馈**：操作后顶部居中显示成功/失败提示，3~5秒自动消失
- **表单校验**：失焦时校验必填项和格式，错误信息显示在字段下方；提交时滚动到第一个错误处
- **加载状态**：首次加载显示骨架屏，提交时按钮显示 loading 禁止双击
