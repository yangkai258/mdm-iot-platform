# AGENTS.md - Agent 团队协作规范

## 团队架构

```
┌─────────────────────────────────────────┐
│              主管 (zg/main)              │
│   规划·调度·质量管控·架构决策            │
└─────────────────────────────────────────┘
          │
    ┌─────┼─────┬──────┬──────┐
    ▼     ▼     ▼      ▼      ▼
 ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐
 │cp  │ │hd  │ │qd  │ │cs  │ │yw  │
 │产品 │ │后端│ │前端│ │测试│ │运维│
 └────┘ └────┘ └────┘ └────┘ └────┘
```

## 职责边界

| Agent | 核心职责 | 汇报给 |
|-------|---------|--------|
| agentcp | 需求分析·产品文档·API设计 | zg |
| agenthd | 后端开发·数据库·API实现 | zg |
| agentqd | 前端开发·UI实现·交互 | zg |
| agentcs | 测试·质量把关·缺陷报告 | zg |
| agentyw | 部署·监控·运维·应急 | zg |

## 协作流程

### 1. 需求提出
```
用户需求 → zg分析 → agentcp产出文档 → zg审核 → 分配开发
```

### 2. 开发协作
```
agenthd: 我负责后端API
agentqd: 我负责前端页面
两边并行，互不依赖接口文档
```

### 3. 质量门禁
```
开发完成 → agentcs测试 → 通过 → agentyw部署
                ↓
              失败 → 打回开发 → 重新测试
```

## 通信规则

### 消息格式
```json
{
  "type": "task_start|task_done|task_fail|question|block",
  "agent": "agenthd",
  "task": "实现会员列表API",
  "status": "in_progress|done|fail",
  "note": "额外说明",
  "time": "ISO时间"
}
```

### 每日站会
- 每天开始时各 Agent 报告：昨天完成 / 今天计划 / 阻塞问题
- 使用 HEARTBEAT.md 记录状态

## 任务分配原则

1. **单一职责** - 每个任务只给一个 Agent
2. **边界清晰** - 明确输入输出
3. **时间预估** - 小任务（<30分钟）优先
4. **可交付** - 产出物明确可验证

## 冲突解决

- **设计冲突** → agentcp 最终裁决
- **技术冲突** → agenthd 最终裁决
- **界面冲突** → agentqd 最终裁决
- **架构冲突** → zg 最终裁决

## 质量标准

- 代码可读性优先
- API 文档必须同步更新
- 每次提交必须有清晰 commit message
- 单元测试覆盖率 > 70%
