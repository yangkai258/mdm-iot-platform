# AGENTS.md - 运维团队协作规范

## 部署规范

### 环境分类
| 环境 | 用途 | 变更权限 |
|------|------|---------|
| dev | 开发测试 | 开发可操作 |
| test | 测试环境 | 测试可操作 |
| pre | 预发布 | 运维操作 |
| prod | 生产环境 | 审批后操作 |

### 发布流程
```
1. 代码合并到主分支
2. 构建镜像
3. 预发布验证
4. 生产发布
5. 监控观察
```

### 回滚流程
```
1. 确认问题影响
2. 启动回滚决策
3. 执行回滚
4. 验证服务正常
5. 通知相关人
```

## 监控规范

### 核心指标
- CPU 使用率 < 80%
- 内存使用率 < 85%
- 磁盘使用率 < 90%
- 接口响应时间 < 200ms
- 错误率 < 1%

### 告警等级
| 等级 | 响应时间 | 说明 |
|------|---------|------|
| P0 | 5分钟 | 服务不可用 |
| P1 | 15分钟 | 部分功能异常 |
| P2 | 1小时 | 性能下降 |
| P3 | 4小时 | 容量预警 |

## 故障处理

### 应急响应
```
发现故障
    ↓
评估影响（30秒）
    ↓
紧急处置（恢复优先）
    ↓
通知相关人
    ↓
定位根因
    ↓
修复验证
    ↓
复盘总结
```

### 沟通模板
```
【故障通报】时间 | 影响范围 | 状态 | 负责人
【故障升级】时间 | 原因 | 需要的帮助
【故障恢复】时间 | 解决方案 | 后续优化
```
