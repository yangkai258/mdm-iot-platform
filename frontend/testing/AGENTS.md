# AGENTS.md - 测试团队协作规范

## 测试策略

### 测试金字塔
```
        /\
       /  \
      / E2E\     <- 少量，关键路径
     /------\
    /集成测试\  <- 核心业务流程
   /----------\
  / 单元测试   \ <- 基础，要多
 /------------\
```

### 测试优先级
1. **P0** - 核心功能（登录、支付等）
2. **P1** - 重要功能（CRUD）
3. **P2** - 一般功能

## 测试用例规范

### 用例格式
```markdown
用例编号: TC-001
用例名称: 会员登录成功
前置条件: 会员已注册
测试步骤:
  1. 输入正确手机号
  2. 输入正确密码
  3. 点击登录
预期结果: 登录成功，跳转到首页
```

### 边界值测试
- 空值
- 最大值/最小值
- 特殊字符
- 格式错误

## 缺陷管理

### 缺陷等级
| 等级 | 定义 | 处理时间 |
|------|------|---------|
| P0 | 系统崩溃、数据丢失 | 立即 |
| P1 | 核心功能不可用 | 24h |
| P2 | 非核心功能异常 | 72h |
| P3 | UI/文案问题 | 下一版本 |

### 缺陷报告
```markdown
缺陷标题: [模块] 缺陷描述
严重程度: P0/P1/P2/P3
复现步骤:
  1. ...
  2. ...
预期结果: ...
实际结果: ...
附件: 截图/日志
```

## 自动化测试

### API 测试
- 使用 Postman/Newman
- 每次部署前跑一遍
- 关键接口加监控

### 回归测试
- 每次发布前执行
- 记录执行结果
- 失败立即通知

---

_测试是质量的守护者，不是开发的绊脚石。_
