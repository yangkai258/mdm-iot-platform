## 一、产品愿景

MDM（移动设备管理）中台是 AI 电子宠物"端云协同"架构的核心骨干，负责统一管理分布在全球的 M5Stack 硬件终端，维护设备全生命周期，并提供会员运营、应用分发、内容管理和策略管控能力。

**核心价值：** 将分散的物理设备接入云端，实现远程监控、固件升级、指令控制、策略合规、应用分发、内容推送、用户运营的完整闭环。

## 三、分阶段规划

### Phase 1：核心能力补齐 + 新模块基础（3周）

#### Sprint 1.1：OTA Worker 实现（P0）
- 实现 `backend/ota/worker.go` 后台轮询部署任务
- 对齐 `models/ota.go` 字段与 PRD 定义
- 注册设备端 OTA 回调路由
- OTA 与应用版本关联支持

#### Sprint 1.2：设备影子修复 + 合规检查（P0）
- 修复 Redis URL 解析失效（生产阻断性bug）
- 设备离线时同步更新 DB lifecycle_status
- 实现 DND 免打扰规则设备端联动
- **集成合规检查 CheckCompliance() 到心跳处理**

#### Sprint 1.3：告警系统增强（P0）
- CheckAlerts 集成到 MQTT heartbeat handler
- **实现越狱/Root 检测告警**
- **实现地理围栏违规告警**
- 设备电量 < 15%、离线 > 90s 自动告警
- 告警确认/解决流程
- **合规策略联动（告警触发 remediation_action）**

#### Sprint 1.4：策略管理基础（P0）
- 策略管理模块 CRUD 实现
- 配置文件库实现（Wi-Fi/VPN/Email/证书/限制）
- 策略绑定功能（设备/用户/组/组织单元）
- 合规规则引擎实现
- 策略版本管理与回滚

## 四、关键依赖关系

```
OTA Worker (P0)
    ↑
    ├─ 依赖 OTA 模型对齐
    ├─ 依赖 MQTT handler
    └─ 依赖设备端固件 OTA 协议支持

设备影子 (P1)
    ↑
    ├─ 依赖 Redis 设备影子存储
    ├─ 依赖 MQTT 心跳上报
    └─ 触发 告警系统 (P1)

策略管理 (P0) [新模块]
    ↑
    ├─ 依赖设备影子（合规检查触发）
    ├─ 触发 告警系统（不合规时）
    └─ 触发 设备影子（desired_config 下发）

告警系统 (P1)
    ↑
    ├─ 依赖设备影子状态
    ├─ 依赖策略管理（越狱/围栏检测）
    └─ 触发 通知管理

应用管理 (P1) [新模块]
    ↑
    ├─ 依赖设备影子（分发指令）
    └─ 触发 通知管理（安装结果）

内容管理 (P1) [新模块]
    ↑
    ├─ 依赖设备影子（分发指令）
    └─ 触发 通知管理

通知管理 (P1) [新模块]
    ↑
    ├─ 依赖设备影子（MQTT 下发）
    ├─ 依赖系统管理（APNs 配置）
    └─ 触发 会员管理（用户通知）

会员管理 (P1)
    ↑
    ├─ 独立模块
    └─ 依赖 会员等级规则引擎

数据分析 (P2)
    ↑
    ├─ 依赖各模块数据积累
    └─ 依赖应用管理（App 统计）
    └─ 依赖策略管理（合规报表）

系统管理 (P2)
    ↑
    └─ 依赖设备影子（设备事件日志）
```

## 六、验收标准总览

### Phase 1 验收

| 模块 | 验收标准 |
|------|----------|
| OTA Worker | 创建部署任务后，Worker 自动选中目标设备并下发 MQTT OTA 指令；成功率 < 80% 自动暂停 |
| 设备影子 | 设备离线 > 90s，DB 中 lifecycle_status 同步更新；DND 时间段设备自动静音；合规检查在心跳中触发 |
| 告警触发 | 设备电量 < 15% 自动创建告警；离线 > 90s 自动创建告警；越狱检测上报后自动创建告警；地理围栏违规自动创建告警 |
| 策略管理 | 配置文件库完整；策略可绑定设备/用户/组/组织单元；不合规时触发对应 remediation_action |

### Phase 2 验收

| 模块 | 验收标准 |
|------|----------|
| 告警通知 | 告警触发后根据 notify_ways 发送邮件/Webhook |
| 应用管理 | 满足 `MODULE_APP_MANAGEMENT.md` 中 P0/P1 功能定义 |
| 通知管理 | 推送通知正确下发到设备；公告正确展示；模板可复用 |
| 内容管理 | 文件正确上传和分发；访问权限正确控制 |
| 会员管理 | 满足 `MODULE_MEMBER_MANAGEMENT.md` 中 P0/P1 功能定义 |
| 积分引擎 | 消费自动增减积分；满足升级阈值自动升级并记录流水 |

### Phase 3 验收

| 模块 | 验收标准 |
|------|----------|
| 应用管理高级 | 强制安装不可卸载；VPP 许可证正确管理 |
| 策略管理高级 | 策略冲突检测正确；合规报表数据准确 |
| 数据分析 | Dashboard 数据准确；App统计/合规报表支持；自定义时间范围筛选 |
| 系统管理 | RBAC 权限正确执行；JWT 密钥从环境变量加载；APNs 正确配置 |
| 组织架构 | 支持 1000+ 员工数据的树形展示和批量操作；身份同步成功 |

### Phase 4 验收

| 模块 | 验收标准 |
|------|----------|
| 数据分析 | 满足 `MODULE_DATA_ANALYSIS.md` 中所有功能 |
| 会员活跃度 | 留存/转化漏斗正确计算 |
| 设备事件日志 | 设备事件正确记录并可查询 |
| 身份同步 | LDAP/SSO 用户正确同步到系统用户 |
