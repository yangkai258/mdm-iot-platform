## 1. 模块定位和目标

### 1.1 模块定位

会员管理模块是 MDM 控制中台的核心业务模块之一，承载会员全生命周期管理、会员营销、积分权益体系等核心零售能力。该模块为运营人员提供完整的会员运营工具，支持多门店统一管理，帮助企业构建会员忠诚度体系。

### 1.2 模块目标

| 目标 | 描述 |
|------|------|
| 会员沉淀 | 建立统一的会员档案，实现全渠道会员数据整合 |
| 精准营销 | 基于会员标签和等级，支持差异化营销策略 |
| 权益驱动 | 通过积分、优惠券、等级权益提升会员活跃度和复购率 |
| 运营提效 | 提供批量打标、自动升级、自动化营销等高效运营工具 |

### 1.3 与其他模块的边界

| 关系模块 | 交互说明 |
|----------|----------|
| 订单模块 | 会员订单数据来源于订单模块，本模块仅做展示和统计 |
| 促销模块 | 促销活动由促销模块配置，本模块负责关联和展示 |
| 店铺模块 | 会员归属门店关系由店铺模块统一管理 |

## 4. 业务流程图（文字描述）

### 4.1 会员注册流程

```
[顾客进店/线上入口]
       ↓
[导购员/顾客自主] → 创建会员档案
       ↓
[录入信息] → 手机号（必填）、姓名、性别、生日、门店
       ↓
[系统分配会员等级] → 根据默认等级规则分配
       ↓
[分配会员卡] → 根据卡类型分配
       ↓
[注册成功] → 发送欢迎短信/优惠券
       ↓
[会员档案生成]
```

### 4.2 会员消费流程

```
[会员到店消费/线上下单]
       ↓
[导购员/顾客自助] → 扫描会员码
       ↓
[系统识别会员] → 查询会员信息、等级、可用优惠券
       ↓
[选择促销/优惠券] → 系统推荐可用优惠
       ↓
[确认订单] → 计算折扣、积分、实付金额
       ↓
[支付完成] → 积分增加、优惠券核销、等级成长值增加
       ↓
[订单完成] → 更新会员消费记录
```

### 4.3 会员升级流程

```
[每日定时任务/消费后触发]
       ↓
[检查会员消费金额] → 统计累计消费
       ↓
[匹配升级条件] → 对比各等级门槛
       ↓
[满足升级条件?] → ─→ 否 → [保持当前等级]
       ↓ 是
[自动升级] → 升级到新等级
       ↓
[发送升级通知] → 短信/小程序通知
       ↓
[更新会员等级]
```

### 4.4 优惠券发放流程

```
[运营人员选择优惠券] → [选择发放会员范围]
       ↓
[选择发放方式] → 定向发放/批量发放/活动发放
       ↓
[确认发放] → 积分兑换/活动奖励/运营手动发放
       ↓
[系统发放优惠券] → 更新优惠券库存和会员卡包
       ↓
[发送通知] → 提醒会员查收
       ↓
[会员使用/过期]
```

### 4.5 积分规则流程

```
[消费完成] → [根据订单金额计算积分]
       ↓
[积分规则配置] → 每消费1元 = N积分（根据等级倍率）
       ↓
[积分入账] → 会员积分增加
       ↓
[积分流水记录] → 记录来源、金额、积分数量
       ↓
[积分使用] → 抵扣订单/兑换优惠券/兑换礼品
       ↓
[积分扣减] → 会员积分减少
```

## 5. 核心场景说明

### 5.1 场景一：会员开卡

**场景描述：** 顾客首次到店，导购员为顾客办理会员卡。

**前置条件：** 门店已开通会员开卡权限

**操作步骤：**
1. 导购员登录系统，进入「会员管理」→「新增会员」
2. 输入顾客手机号（系统自动检索是否已存在）
3. 若手机号已存在，提示「该手机号已是会员，是否查看详情」
4. 若手机号不存在，填写会员信息：姓名、性别、生日、所属门店
5. 选择会员卡类型（默认根据门店配置）
6. 确认信息无误后点击「确定」
7. 系统分配会员编号，发送开卡成功短信

**异常处理：**
- 手机号格式错误：提示「请输入正确的手机号」
- 门店未配置默认卡类型：提示「请联系管理员配置门店卡类型」

**业务规则：**
- 手机号必须唯一
- 会员编号自动生成，格式：HY+日期+6位序号
- 开卡后自动同步至微信会员卡（如已绑定）

---

### 5.2 场景二：会员等级调整

**场景描述：** 会员累计消费达到升级条件，系统自动升级；或运营人员手动调整会员等级。

**自动升级规则：**
| 累计消费金额 | 可升级至 |
|--------------|----------|
| 0 - 999元 | 普通会员 |
| 1000 - 4999元 | 银卡会员 |
| 5000 - 9999元 | 金卡会员 |
| 10000元以上 | 钻石卡会员 |

**手动调整操作步骤：**
1. 运营人员进入「会员管理」→「会员列表」
2. 搜索目标会员，点击「编辑」
3. 修改会员等级
4. 选择调整原因（下拉选项：主动降级、投诉补偿、特殊照顾、其他）
5. 填写备注说明
6. 确认修改

**降级规则：**
- 连续12个月无消费，自动降一级
- 降级前7天发送预警通知

---

### 5.3 场景三：优惠券发放与核销

**发放场景：**
1. **新会员开卡礼：** 开卡即送指定优惠券
2. **生日礼包：** 会员生月赠送专属优惠券
3. **活动奖励：** 参与活动后发放奖励优惠券
4. **定向营销：** 筛选会员群体后批量发放
5. **积分兑换：** 会员使用积分兑换优惠券

**核销场景：**
- 顾客到店消费时出示优惠券码
- 导购员扫描优惠券二维码
- 系统验证优惠券有效性（未使用、未过期、状态正常）
- 验证通过后核销，关联订单

**有效期规则：**
- 固定有效期：设置具体开始和结束日期
- 领取后生效：领取之日起N天内有效
- 券类别支持：满减券、折扣券、兑换券

---

### 5.4 场景四：积分兑换

**积分获取规则（示例）：**
| 行为 | 积分倍数 |
|------|----------|
| 消费（普通会员） | 1元 = 1积分 |
| 消费（银卡会员） | 1元 = 1.5积分 |
| 消费（金卡会员） | 1元 = 2积分 |
| 消费（钻石卡会员） | 1元 = 3积分 |
| 生日双倍 | 生日当天消费积分翻倍 |
| 活动奖励 | 参与活动获得额外积分 |

**积分抵扣规则：**
- 100积分 = 1元（可配置）
- 单笔订单最高抵扣50%订单金额
- 积分不足100时不可抵扣

**积分使用场景：**
1. 订单抵扣：结算时选择使用积分
2. 积分兑换：兑换优惠券、礼品
3. 积分抽奖：参与积分抽奖活动

---

### 5.5 场景五：批量打标

**场景描述：** 运营人员需要为符合特定条件的会员批量添加标签。

**操作步骤：**
1. 进入「会员标签」→「批量打标」
2. 选择标签类型（单选）
3. 设置筛选条件：
   - 会员等级
   - 累计消费区间
   - 注册时间区间
   - 最近消费时间
   - 所属门店
4. 点击「筛选」，预览符合条件的会员数量
5. 确认无误后点击「确认打标」
6. 系统后台处理，完成后提示

**业务规则：**
- 单次批量打标上限10000人
- 已拥有该标签的会员会自动跳过
- 操作记录留存，支持追溯

---

### 5.6 场景六：促销活动应用

**促销类型：**

1. **买赠促销**
   - 示例：购买A商品满2件，赠B商品
   - 规则：指定商品达到数量后自动添加赠品

2. **直减促销**
   - 示例：指定商品立减20元
   - 规则：结算时自动减免

3. **满额减促销**
   - 示例：满200减30、满500减100
   - 规则：订单金额达到门槛自动减免

4. **满额折促销**
   - 示例：满300享9折、满500享8折
   - 规则：订单金额达到门槛自动打折

**促销叠加规则：**
- 同一类型促销不可叠加（取最优）
- 优惠券与促销活动可叠加使用
- 会员折扣与促销活动不可叠加

## 6. 数据规范

### 6.1 会员编号规则

格式：`HY + YYYYMMDD + 6位序号`
示例：`HY20260319000001`

### 6.2 会员状态

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| 1 | 正常 | 可正常消费、使用积分优惠券 |
| 2 | 冻结 | 临时限制，冻结期间不可消费 |
| 3 | 禁用 | 永久限制，通常因违规行为 |

### 6.3 会员等级

| 等级ID | 等级名称 | 英文标识 | 升级门槛（累计消费） |
|--------|----------|----------|---------------------|
| 1 | 普通会员 | NORMAL | 0元 |
| 2 | 银卡会员 | SILVER | 1000元 |
| 3 | 金卡会员 | GOLD | 5000元 |
| 4 | 钻石会员 | DIAMOND | 10000元 |

### 6.4 标签类型

| 类型ID | 类型名称 | 说明 |
|--------|----------|------|
| 1 | 属性标签 | 会员固有属性，如：高价值、沉睡用户 |
| 2 | 行为标签 | 基于消费行为生成，如：高频购买、凑单用户 |
| 3 | 自定义标签 | 运营自定义创建的标签 |
