## 十七、数据库设计

> **版本号：** V2.0  
> **更新日期：** 2026-03-21

### 17.1 数据库选型与配置

| 配置项 | 说明 |
|--------|------|
| **数据库类型** | PostgreSQL 15+ |
| **ORM 框架** | GORM v2 |
| **连接池** | MaxOpenConns=100, MaxIdleConns=20, ConnMaxLifetime=1h |
| **字符集** | UTF-8 (en_US.UTF-8) |
| **时区** | Asia/Shanghai (UTC+8)，所有存储字段以 TIMESTAMPTZ 为主，页面展示时转换 |
| **审计字段** | 所有表含 created_at (timestamptz)、updated_at (timestamptz)、deleted_at (软删，gorm.delegated) |

### 17.2 多租户隔离策略

- **隔离模式**：共享数据库 + 独立 Schema（租户 UUID 为 Schema 名）
- **行级隔离**：核心业务表含 tenant_id 字段，所有查询默认以 tenant_id 过滤
- **中间件拦截**：GORM Callback 自动注入 tenant_id 条件，禁止跨租户访问
- **超管视图**：超管查询不注入 tenant_id，可跨租户汇总

### 17.3 核心数据表

#### 17.3.1 租户主表（tenants）

```sql
CREATE TABLE tenants (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_code     VARCHAR(30) UNIQUE NOT NULL,        -- 租户编码，系统唯一
    tenant_name     VARCHAR(100) NOT NULL,              -- 租户全称
    tenant_short_name VARCHAR(50),                       -- 简称
    logo_url        VARCHAR(500),                        -- logo URL
    contact_name    VARCHAR(50),                         -- 联系人姓名
    contact_phone   VARCHAR(20),                         -- 联系电话
    contact_email   VARCHAR(100),                        -- 联系邮箱
    status          VARCHAR(20) DEFAULT 'pending' CHECK (status IN ('pending','active','frozen','cancelled')),
    approved_at     TIMESTAMPTZ,                         -- 审核通过时间
    approved_by     UUID,                                -- 审核人
    expires_at      TIMESTAMPTZ,                         -- 到期时间（NULL=永不过期）
    plan_id         UUID,                                -- 当前套餐 ID
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ
);

CREATE INDEX idx_tenants_status ON tenants(status);
CREATE INDEX idx_tenants_plan ON tenants(plan_id);
```

### 17.4 数据库规范

| 规范项 | 要求 |
|--------|------|
| **主键** | 全部使用 UUID，程序层生成 |
| **索引** | 外键字段、常用查询字段必须建索引 |
| **命名** | 表名单数，snake_case；字段 snake_case；索引 idx_{表名}_{字段} |
| **软删除** | 所有业务表使用 gorm.deletedAt，支持软删恢复 |
| **敏感数据** | 密码加密（bcrypt），MQTT 密码加密（AES） |
| **JSON 字段** | 使用 JSONB 类型存储扩展属性，支持索引查询 |
| **时区** | 所有 TIMESTAMPTZ 存储 UTC，Go 层转换后输出 RFC3339 |

### 18.3 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": { ... },
  "request_id": "uuid-v4",
  "timestamp": "2026-03-21T10:30:00Z"
}
```

| code | 说明 |
|------|------|
| 0 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证或 Token 过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 429 | 请求过于频繁（限流） |
| 500 | 服务器内部错误 |

router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  if (!authStore.token && to.path !== '/login') {
    return next('/login')
  }

  if (!authStore.hasPermission(to.meta.permission)) {
    return next('/403')
  }
  next()
})

## 二十、非功能性需求

> **版本号：** V2.0  
> **更新日期：** 2026-03-21

### 20.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 页面首屏加载 | ≤ 2s（4G 网络） | FCP ≤ 2s，LCP ≤ 3s |
| API 响应时间 | P95 ≤ 500ms | 99% 接口在 500ms 内响应 |
| 查询接口响应 | P95 ≤ 1s | 列表查询、统计接口 |
| 并发用户数 | ≥ 500 租户同时在线 | 单租户 10 用户并发 |
| 设备数据上报 | ≤ 100ms 延迟 | MQTT 消息延迟 |
| OTA 下载速度 | ≥ 10 MB/s | 单设备固件下载速率 |
| 设备心跳处理 | ≥ 10000 条/秒 | Redis 写吞吐 |

### 20.2 可用性需求

| 指标 | 要求 |
|------|------|
| 系统可用性 | ≥ 99.9%（月停机 ≤ 43min） |
| 年度正常运行时间 | ≥ 360 天 |
| 故障恢复时间（MTTR） | ≤ 30 分钟 |
| 数据备份周期 | 每日全量，支持按需增量 |
| 备份保留周期 | 最近 30 天 |

### 20.3 安全需求

| 类别 | 要求 |
|------|------|
| 认证 | 所有 API 需 JWT Token 认证，超管与租户 Token 分离 |
| 密码存储 | bcrypt（cost=12），禁止明文 |
| 传输加密 | 全站 HTTPS（TLS 1.2+） |
| SQL 注入 | ORM 参数化查询，禁止拼接 SQL |
| XSS | 前后端双向内容转义 |
| CSRF | Token 验证，SameSite=Strict |
| 敏感配置 | 加密存储，运行时解密 |
| 日志脱敏 | 日志中密码、Token 字段自动掩码 |
| 接口限流 | 基于 IP + UserID 双重限流 |
| 审计日志 | 关键操作（增删改）全量记录 |

### 20.4 扩展性需求

| 维度 | 要求 |
|------|------|
| 水平扩展 | 后端服务无状态，支持多实例部署 |
| 数据库扩展 | 支持读写分离，租户 Schema 模式便于分库 |
| 缓存扩展 | Redis Cluster 支持，支持多租户缓存 Key 前缀隔离 |
| 消息队列 | MQTT + Kafka 双轨，OTA 推送与日志采集解耦 |
| 插件机制 | 未来支持业务模块插件化注册 |
